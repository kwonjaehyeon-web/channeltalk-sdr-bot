# PRD: 세일즈 인텔리전스 플랫폼

**버전 1.0 | 2026-03-20 | 상태: 초안**

---

## Gemini 제안 검토: 핵심 보완점

| 영역 | Gemini의 한계 | 개선안 |
|------|------------|-------|
| 아키텍처 | 단순 크롤링→LLM 직결 | 비동기 작업 큐(Celery+Redis) 필수 |
| 모델 전략 | 단일 LLM으로 모든 작업 | 작업 난이도별 모델 분리 (비용 80% 절감 가능) |
| 법적 리스크 | LinkedIn 크롤링 언급 | ToS 위반 - MVP에서 완전 제외 |
| 데이터 신선도 | 언급 없음 | 수집일 태깅 + TTL 만료 처리 필수 |
| 출력 | Notion 우선 | Google Sheets 우선 (Notion은 API 복잡도+비용) |
| 한국 특화 | 나라장터 언급 누락 | 공공기관 조달 이력 P1으로 추가 |

## 1. 목표 및 배경

### 배경

한국 B2B 영업에서 고객사 사전 리서치에 **평균 2-4시간** 소요. 채용공고는 기업이 돈을 써서라도 해결하고 싶은 "결핍"이 드러나는 곳이자, **예산 집행 의지와 조직 변화의 가장 빠른 공개 신호**다.

### 목표

- 영업 담당자 1인당 리서치 시간: **2시간 → 15분**
- 주관적 판단 → 데이터 기반 신호 전환
- 팀 인텔리전스 자산 축적

### 핵심 가설

> IT/디지털 전환 채용이 급증하는 기업은 향후 6개월 내 솔루션 구매 가능성이 높다.

|---------|---------|

| SDR/BDR | "콜드 이메일 100개를 각 기업에 맞춤화해서 보내고 싶다" |

## 3. 기능 요구사항

### P0 — MVP 필수

**F-01: 기업 프로파일 자동 생성**
- 기업명 입력 → 10분 내 프로파일 생성
- 포함: 기업 개요, 채용 동향, 조직 규모 추정, 공시 이벤트
- 수집일 기준 30일 초과 데이터는 경고 표시 필수

**F-02: 채용 신호 분석**
- 사람인/잡코리아/원티드 채용공고 수집
- IT/디지털 채용 급증 탐지 (전월 대비 20% 이상)
- 채용 직군 자동 분류 (기술/관리/영업/운영)
- CTO/CDO/IT팀장 채용 시 플래그

**F-03: 공시 이벤트 모니터링**
- DART 전자공시 API 연동
- 이벤트 분류: M&A, 신규투자, 조직개편, 임원변경, 자금조달
- 솔루션 연관성 자동 태깅

**F-04: 영업 신호 스코어링**
- 기업별 구매 가능성 점수 (0-100)
- 신호 기여도 설명 포함
- 시계열 스코어 변화 추적

### P1 — 1차 고도화

**F-05: 의사결정권자 매핑**
- 공개 데이터 기반 키맨 식별
- 주의: 개인 연락처 수집 금지, 포지션/역할 정보만 허용

**F-06: 경쟁 인텔리전스**
- 경쟁사 도입 신호 탐지 (채용공고 우대사항 기반)
- 화이트스페이스 기업 (미도입) 식별

**F-07: 맞춤 어프로치 메시지 생성**
- 채널별 초안: 이메일 / LinkedIn DM / 전화 스크립트

**F-08: 모니터링 & 알림**
- 관심 기업 변화 발생 시 Slack/이메일 알림
- 알림 빈도 설정 (실시간/일간/주간)

### P2 — 2차 고도화

**F-09: ICP 기반 타겟 자동 발굴** — 이상적 고객 프로파일 정의 후 매칭 기업 추천

**F-10: 팀 협업** — 노트, 담당자 할당, 중복 접촉 방지

**F-11: CRM 연동** — HubSpot/Salesforce 양방향 동기화

**F-12: 나라장터 조달 이력** — 공공기관 영업용 계약 만료 타이밍 알림

## 4. 기술 아키텍처

### 전체 구조

```
[데이터 수집]          [처리]                  [서비스]
┌───────────┐        ┌────────────────┐      ┌──────────────┐
│ 채용 크롤러 │──────► │ 메시지 큐       │ ──►  │ API 서버      │
│(Playwright)│        │ (Celery+Redis) │      │ (FastAPI)    │
├───────────┤        ├────────────────┤      ├──────────────┤
│ DART API  │──────► │ 정규화 파이프라인 │      │ LLM 오케스트  │
│ (공식 API) │        ├────────────────┤      │ 레이션 엔진   │
├───────────┤        │ 신호 감지기     │      ├──────────────┤
│ Serper API│──────► │ (규칙+LLM 혼합) │      │ 알림 서비스   │
└───────────┘        └────────────────┘      └──────────────┘
                              │                      │
                    ┌─────────▼──────┐    ┌─────────▼──────┐
                    │ PostgreSQL     │    │ 외부 연동        │
                    │ + pgvector     │    │ CRM/Slack/Sheets│
                    └────────────────┘    └────────────────┘
```

### 데이터 소스 우선순위

| 소스 | 데이터 | 수집 방식 | 법적 리스크 | 우선순위 |
|------|--------|---------|-----------|---------|
| DART 전자공시 | 사업보고서, 공시 이벤트 | 공식 Open API | 없음 | P0 |
| 사람인/잡코리아 | 채용공고 | Playwright | 중간 | P0 |
| 원티드 | 채용공고 | 공개 API | 낮음 | P0 |
| Serper | 뉴스, 기업 정보 | API | 낮음 | P0 |
| 나라장터 | 조달 이력 | 공식 Open API | 없음 | P1 |
| 공공데이터포털 | 사업자 정보 | 공식 API | 없음 | P1 |
| LinkedIn | 조직구조 힌트 | ~~크롤링~~ | **높음 - 제외** | 제외 |

### LLM 오케스트레이션: 단계별 모델 분리

```
Step 1: 구조화 추출 (분류, 키워드)
  → 소형 모델 (GPT-4o-mini / Gemini Flash)  ← 비용 최적화

Step 2: 신호 해석 (채용 패턴 → 조직 변화)
  → 중형 모델 (GPT-4o / Gemini Pro)

Step 3: 전략 인사이트 + 어프로치 메시지
  → 고성능 모델 (Claude Sonnet / GPT-4o)

Step 4: 품질 검증 (할루시네이션 필터링)
  → 규칙 기반 체크 + 검증 프롬프트
```

> 단일 모델 의존 금지: OpenAI + Gemini 동시 지원, 장애 시 자동 fallback

### 온디맨드 프로파일 생성

```

    │

    ├── 채용 크롤러: 사람인/잡코리아/원티드
    ├── DART API: 최근 2년 보고서
    └── Serper: 최근 6개월 뉴스
    │
    ▼ [정규화 파이프라인]
    ├── 채용: 직군 분류, 키워드 추출, 날짜 정규화

    │
    ▼ [신호 감지 엔진]
    ├── 채용 급증 감지 (통계 분석)
    ├── IT/DX 전환 신호 식별
    └── 조직 변화 신호
    │
    ▼ [LLM 인사이트 생성]
    ├── 조직 변화 해석
    ├── 예산 가용성 추정
    ├── 영업 진입 포인트 추천
    └── 어프로치 메시지 초안
    │
    ▼ 리포트 저장 + 사용자에게 전달
```

```
    │
    ▼ [변화 감지기]
```

## 6. MVP 범위 (4주 목표)

### 포함

- DART + 사람인 + Serper 수집
- 채용 트렌드 분석 (직군 분류, 월별 증감)
- 영업 신호 스코어 계산
- 기업별 1페이지 인텔리전스 카드
- 어프로치 메시지 초안 1개
- Google Sheets 내보내기
- UI: Streamlit 단일 페이지

### 제외

- 실시간 모니터링 (주 1회 배치로 제한)
- 팀 협업 기능
- CRM 연동
- LinkedIn 데이터

### 성공 지표

- 프로파일 생성 시간 10분 이내
- 파일럿 사용자 70% 이상 "유용함" 평가
- 파일럿 참여자 50% 이상 주 1회 사용

| Phase | 기간 | 목표 | 체크포인트 |
|-------|------|------|-----------|
| **Phase 0** | 1-2주 | 데이터 파이프라인 뼈대 | 기업명 입력 → 원시 데이터 수집 완료 |

## 8. 리스크 관리

### 법적/윤리 리스크 (최우선)

| 리스크 | 대응 |
|--------|------|
| 개인정보보호법(PIPA) | 개인 연락처 수집 절대 금지. 직책/역할/공개 활동만 허용 |
| 크롤링 ToS 위반 | robots.txt 자동 감지, 사이트별 Rate Limiting 철저 준수 |
| LinkedIn 크롤링 | MVP에서 완전 제외 (ToS 위반 + 법적 불확실성) |
| DART 데이터 재배포 | 내부 도구 용도로만 사용, 이용약관 사전 확인 필수 |

### 기술 리스크

| 리스크 | 대응 |
|--------|------|
| 크롤러 DOM 변경으로 중단 | 사이트별 크롤러 독립 모듈, 변경 감지 알림 |
| LLM 할루시네이션 | 모든 인사이트에 근거 데이터 출처 명시, 추정/확인 구분 |
| 데이터 오염 (오래된 정보) | 수집일 태깅 + TTL 만료, UI에 신선도 표시 |
| LLM 비용 폭증 | 변화 없는 기업은 분석 Skip, 기본 감지는 규칙 기반 |

## 9. 한국 B2B 특화 신호 패턴

### 고신뢰 신호 (스코어 높게 부여)

1. "DX 추진단" / "디지털 전환 TF" 조직 신설 채용
2. CTO/CDO 포지션 **신규** 채용 (기존에 없던 경우)
3. DART 공시: 100억 이상 IT 인프라 투자 결정
4. 기업 분할/자회사 설립 직후 (새 조직 = 새 예산)
5. 대표이사/C레벨 교체 후 90일 이내

### 중간 신뢰 신호

6. AWS/Azure 관련 채용 급증 (클라우드 전환 예산 신호)
7. IT 직군 채용 전월 대비 30% 이상 증가
8. 스타트업/테크 출신 임원 영입 (도입 의사결정 빠름)

### 약한 신호 (보조 지표로만 사용)

9. 기업 홈페이지 리뉴얼
10. IT 관련 콘퍼런스 스폰서십
