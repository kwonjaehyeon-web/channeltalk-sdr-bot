# PRD: 기업 분석 자동화 슬랙 봇 (Biz-Scouter)

---

## 1. 프로젝트 개요 (Overview)

**목적**: 영업 및 비즈니스 미팅 전, 상대 기업의 의사결정권자, 조직의 힘(Power Center), 서비스 취약점을 1분 내로 요약하여 슬랙 리포트로 제공.

**타겟 유저**: B2B 영업 사원, 전략 기획자, 투자 심사역.

**핵심 가치**: DART의 정량 데이터와 채용/뉴스 기반의 정성 데이터를 결합하여 "누구에게, 무엇을 제안할지" 전략 도출.

---

## 2. 주요 기능 (Key Features)

### ① 멀티 소스 데이터 수집 (Data Aggregation)

**DART (상장사)**
- 최근 공시 목록: M&A, 자금조달, 임원변경, 조직개편
- 분기/사업보고서: 매출·영업이익 추이, 부서별 임직원수
- 등기 임원 현황: 대표이사·사내이사 이름 및 변경 이력

**Serper (뉴스/검색)**
- 최근 6개월 주요 뉴스
- 인터뷰 기사 기반 실무 결정권자 이름 추출
- `"기업명 임직원수"` 검색으로 규모 추정
- `"기업명 DX OR 클라우드 OR AI"` 검색으로 전환 현황 파악

**채용 사이트 (사람인/잡코리아)**
- 직군별 채용 공고 수 분포 → 어떤 부서에 예산이 쏠리는가
- 채용 JD의 '담당 업무' 텍스트에서 조직 키워드 추출
- 우대사항의 기술스택 → 현재 사용 중인 경쟁사 제품 탐지
- 관리자급(팀장↑) 신규 채용 여부 → 의사결정권자 변화 신호

**비상장사 대응**
- DART 데이터 부재 시 Serper + 채용공고만으로 분석
- 크레딧잡/잡플래닛 인원수 정보 보완 (Serper 검색)
- LLM에게 "비상장사, 실데이터 제한적" 컨텍스트 명시 → 할루시네이션 방지

### ② AI 전략 분석 (AI Insights)

**LLM 선택: GPT-4o (기본) / Claude 3.5 Sonnet (대안)**
- GPT-4o: JSON 출력 안정성 높음, response_format 지원
- Claude: 한국어 뉘앙스, 전략 추론에 강점 — 추후 전환 고려

**도출하는 6가지 핵심 인사이트**

| 항목 | 설명 | 영업 활용 |
|------|------|----------|
| 구매 신호 스코어 | 0~100점, 근거 포함 | 우선순위 판단 |
| ICP 적합도 | high/medium/low + 산업군/매출/SaaS 가능성 | 타겟 여부 즉시 판단 |
| Pain Points | 3가지 + 근거 + 대응 솔루션 카테고리 | 제안 포인트 |
| Qualify 질문 | 미팅에서 던질 4가지 질문 | 실무 도구 |
| SaaS 추천 | 카테고리별 제품명 + 추천 이유 | 크로스셀/업셀 |
| 어프로치 메시지 | Pain 중심 첫 연락 초안 | 콜드 아웃리치 |

### ③ 슬랙 인터페이스 (Slack UI)

**커맨드**
- `/회사 [기업명]` → 즉시 분석
- `/회사 watch [기업명]` → 워치리스트 등록 (변화 시 자동 알림)
- `/회사 list` → 내 워치리스트 확인

**리포트 포맷 (Block Kit)**

## 3. 스코어링 알고리즘

### 구매 신호 스코어 (0~100)

| 신호 | 가중치 | 데이터 소스 |
|------|--------|------------|
| DX본부/디지털전환TF 신설 채용 | +30 | 채용공고 |
| CTO/CDO 신규 채용 (기존 없던 포지션) | +25 | 채용공고 |
| IT 인프라 100억↑ 투자 공시 | +25 | DART |
| 기업 분할/자회사 설립 | +20 | DART |
| 대표이사 교체 90일 이내 | +20 | DART |
| AWS/Azure 관련 채용 급증 | +15 | 채용공고 |
| IT 직군 채용 전월比 30%↑ | +12 | 채용공고 |
| DX/클라우드 관련 뉴스 다수 | +10 | Serper |
| 스타트업/테크 출신 임원 영입 | +10 | Serper |
| 최대 cap | 100 | |

> 신호가 없으면 OpenAI 학습 데이터 기반 추정값 사용 (추정임을 명시)

## 4. 데이터 워크플로우 (Workflow)

```
① Trigger
   사용자가 /회사 [기업명] 입력
        │
        ▼
② 캐시 확인 (24시간)
   있으면 → 즉시 리포트 반환
   없으면 → 수집 시작
        │
        ▼
③ 병렬 수집 (3~5분 소요)
   ├── DART API     → 공시 이벤트, 임직원수, 임원 현황
   ├── Serper API   → 뉴스, 규모 추정, DX 현황
   └── 사람인 크롤링 → 채용 공고 목록, 직군 분포
        │
        ▼
④ 데이터 정규화
   - 채용공고: 직군 분류, 전월比 증감율 계산
   - DART: 이벤트 분류 (investment/ma/executive/restructure)
   - 뉴스: 날짜 필터링 (6개월 이내만), 키워드 태깅
        │
        ▼
⑤ 신호 감지 + 스코어 계산
   - 각 신호별 가중치 합산
   - 상장사/비상장사 분기 처리
        │
        ▼
⑥ LLM 인사이트 생성 (Claude 3.5 Sonnet)
   - 수집 데이터 + 스코어를 프롬프트로 전달
   - Power Center / Pain Points / Approach Strategy 도출
   - 추정 정보는 반드시 "(추정)" 레이블 표시
        │
        ▼
⑦ 슬랙 리포트 전송 (Block Kit)
   + DB 캐시 저장 (24시간)
```

### 비상장사 분기 처리
```
DART 기업코드 없음
    │
    ▼
Serper로 보완
  - "기업명 임직원수" 검색
  - "기업명 매출" 검색
  - 크레딧잡/잡플래닛 스니펫 파싱
    │
    ▼
LLM 프롬프트에 명시:
  "이 기업은 비상장사로 DART 데이터가 없습니다.
   검색 결과만으로 분석하며, 불확실한 수치는 (추정)으로 표기하세요."
```

## 5. 기술 스택 (Tech Stack)

| 영역 | 선택 | 이유 |
|------|------|------|
| Language | Python 3.11+ | |
| Slack | Slack Bolt (Python) | Socket Mode로 서버 불필요 |
| LLM | Claude 3.5 Sonnet → GPT-4o fallback | Claude가 한국어 전략 추론에 강점 |
| 공시 | DART Open API | 공식, 무료, 안정적 |
| 검색 | Serper API | 구글 결과를 JSON으로 반환 |
| 크롤링 | Playwright (사람인) | 동적 페이지 대응 |
| HTML 파싱 | BeautifulSoup | 정적 파싱 보조 |
| 비동기 | Python threading | Celery 없이 경량 처리 |
| 캐시 | 파일 캐시 (JSON) → 추후 Redis | MVP는 파일로 충분 |
| DB | SQLite → 추후 PostgreSQL | MVP는 SQLite로 충분 |
| 배포 | Docker + EC2 또는 Railway | Socket Mode라 서버리스 불가 |

> **Socket Mode 주의**: Slack Socket Mode는 상시 연결 프로세스가 필요하므로 Vercel/Lambda 같은 서버리스 환경에서는 동작하지 않음. EC2, Railway, Render 등 상시 실행 서버 필요.

## 6. 에러 처리 및 데이터 품질

### 소스별 실패 처리
```
DART 실패   → Serper + 채용공고만으로 분석 계속
Serper 실패 → DART + 채용공고만으로 분석 계속
크롤링 실패 → 채용 신호 없이 분석 계속 (리포트에 "채용 데이터 수집 실패" 표시)
전부 실패   → LLM 학습 데이터 기반으로만 분석 (리포트에 "실데이터 없음" 경고)
```

### 데이터 신선도
- 모든 섹션에 수집 날짜 표시
- 30일 이상 된 캐시는 재수집 권고 메시지 표시
- 뉴스는 6개월 이내 데이터만 사용

### 할루시네이션 방지
- LLM 프롬프트에 "근거 없는 수치 생성 금지" 명시
- 실데이터 기반 내용과 추정 내용을 구분해서 출력하도록 강제
- 임원 이름은 DART 또는 뉴스 출처 있을 때만 표시

- [ ] 사람인 크롤링
